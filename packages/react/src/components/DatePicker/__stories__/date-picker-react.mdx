# Date Picker (React)

The Date Picker component allows users to select dates using a calendar interface or manual input. This React implementation maintains 100% backwards compatibility with Carbon React v11 while using a modern state machine architecture shared with Web Components.

## Overview

The Date Picker consists of two main components:
- **DatePicker**: The wrapper component that manages state and calendar visibility
- **DatePickerInput**: The input field component with Carbon styling

## Key Features

- ✅ **100% Carbon React v11 Compatible**: Drop-in replacement with no breaking changes
- ✅ **Shared State Machine**: Same logic as Web Components implementation
- ✅ **Three Modes**: Single date, date range, and simple (no calendar)
- ✅ **Validation States**: Invalid, warning, and helper text support
- ✅ **Accessibility**: Full keyboard navigation and ARIA support
- ✅ **Temporal API**: Modern date handling with Temporal.PlainDate
- ✅ **TypeScript**: Full type safety throughout

## Controlled Component

<Canvas of={DatePickerStories.Controlled} />

```tsx
function ControlledExample() {
  const [value, setValue] = useState('');

  return (
    <DatePicker
      onChange={(dates) => {
        if (dates.length > 0) {
          const date = dates[0];
          setValue(`${date.getMonth() + 1}/${date.getDate()}/${date.getFullYear()}`);
        }
      }}
    >
      <DatePickerInput
        value={value}
        onChange={(e) => setValue(e.target.value)}
      />
    </DatePicker>
  );
}
```

## Architecture

### Shared State Machine

The React implementation uses the same state machine as the Web Components version, ensuring:
- Consistent behavior across frameworks
- Single source of truth for date logic
- Easier testing and maintenance
- Framework-agnostic core

### Temporal API

Uses the modern Temporal API for robust date handling:
- No timezone issues
- Precise date calculations
- Better performance
- Future-proof

## Best Practices

### Do's ✅
- Always provide a unique `id` for each DatePickerInput
- Use `labelText` for accessibility
- Provide `helperText` for additional context
- Use validation states (`invalid`, `warn`) for user feedback
- Set `minDate` and `maxDate` to constrain selectable dates

### Don'ts ❌
- Don't use inline styles (use Carbon classes)
- Don't forget to handle the `onChange` callback
- Don't mix controlled and uncontrolled patterns
- Don't use deprecated date formats

## Examples

### With Form Integration
```tsx
function MyForm() {
  const [formData, setFormData] = useState({ date: '' });

  return (
    <form onSubmit={handleSubmit}>
      <DatePicker
        onChange={(dates) => {
          setFormData({ ...formData, date: dates[0]?.toISOString() });
        }}
      >
        <DatePickerInput
          id="form-date"
          labelText="Select Date"
          required
        />
      </DatePicker>
      <button type="submit">Submit</button>
    </form>
  );
}
```

### With Validation
```tsx
function ValidatedDatePicker() {
  const [date, setDate] = useState('');
  const [error, setError] = useState('');

  const validate = (dates: Date[]) => {
    if (dates.length === 0) {
      setError('Date is required');
      return false;
    }
    if (dates[0] < new Date()) {
      setError('Date must be in the future');
      return false;
    }
    setError('');
    return true;
  };

  return (
    <DatePicker onChange={(dates) => validate(dates) && setDate(dates[0].toISOString())}>
      <DatePickerInput
        id="validated-date"
        invalid={!!error}
        invalidText={error}
      />
    </DatePicker>
  );
}
```
