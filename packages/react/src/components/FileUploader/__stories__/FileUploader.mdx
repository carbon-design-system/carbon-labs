## Composable Examples

The FileUploader component provides flexible, composable building blocks that
allow you to create custom file upload experiences. These examples demonstrate
how to use `FileUploaderButton` and `FileUploaderItem` components independently
to build tailored solutions.

### Why Use Composable Components?

The composable approach gives you:

- **Full control** over layout and styling
- **Custom file validation** logic
- **Flexible file item rendering** with your own components
- **Integration** with forms and state management
- **Reusable patterns** across your application

### Available Components

#### FileUploaderButton

A button component that triggers file selection. It handles:

- File input management
- Multiple file selection
- File type restrictions
- Accessibility features

#### FileUploaderItem

A pre-built file item component with:

- File name display
- Status indicators (edit, invalid)
- Error messaging
- Delete functionality
- Size variants

### Implementation Patterns

#### Basic Pattern

```jsx
import { FileUploaderButton, FileUploaderItem } from '@carbon-labs/react';

function MyFileUploader() {
  const [files, setFiles] = useState([]);
  const buttonRef = useRef(null);

  const handleChange = (event) => {
    const fileList = event.target.files;
    setFiles(Array.from(fileList));
  };

  const handleDelete = (fileName) => {
    const updated = files.filter((f) => f.name !== fileName);
    setFiles(updated);
    buttonRef.current?.setFiles(updated);
  };

  return (
    <>
      <FileUploaderButton
        ref={buttonRef}
        onChange={handleChange}
        accept=".jpg,.png"
        multiple
      />
      {files.map((file) => (
        <FileUploaderItem
          key={file.name}
          name={file.name}
          status="edit"
          onDelete={() => handleDelete(file.name)}
        />
      ))}
    </>
  );
}
```

#### With Validation

```jsx
const MAX_SIZE = 512 * 1024; // 512KB

const validateFile = (file) => {
  return file.size <= MAX_SIZE;
};

// In your render:
<FileUploaderItem
  name={file.name}
  status={validateFile(file) ? 'edit' : 'invalid'}
  errorSubject="File too large"
  errorBody="Maximum file size is 512KB"
  onDelete={() => handleDelete(file.name)}
/>;
```

#### Form Integration

```jsx
<Form onSubmit={handleSubmit}>
  <FileUploaderButton name="attachments" onChange={handleChange} />
  {/* Render file items */}
  <Button type="submit">Submit</Button>
</Form>
```

### Best Practices

1. **Always use a ref** with FileUploaderButton to sync internal state when
   deleting files
2. **Provide unique keys** for file items (e.g., `${file.name}-${file.size}`)
3. **Validate files** before rendering to provide immediate feedback
4. **Clear error messages** help users understand what went wrong
5. **Accessible labels** ensure screen reader users can interact with the
   component

### Customization Tips

- Use the `size` prop consistently across button and items for visual harmony
- Customize error messages to match your validation rules
- Consider using `hasIconOnly` for compact layouts
- Wrap in appropriate containers (Tile, Modal, etc.) for context
- Add loading states during file uploads for better UX
