# Guidelines  

This document outlines the best practices and rules for writing MDX documentation 
for components under `@carbon-labs`. 

Following these guidelines will ensure all documented components have the same styles
as the ones found in the [Carbon Design System](https://carbondesignsystem.com) page.


## Meta component
- The `Meta` Storybook MDX component must be included at the top of the file with a `title`. This allows you to determine where the file is included in the side navigation, instead of it being placed based on directory path by default.  
  ```mdx
  <Meta title="ComponentName/Example" />
  ```

## Unsupported features  
This library does **not** support the following MDX features:
- **Frontmatter** (metadata at the beginning of the file)
- **HTML comments** (e.g., `<!-- comment -->`)
- **Markdown image syntax** (e.g., `![alt text](./image.png)`)

## Using images  
Instead of markdown image syntax, import images and use the `<img />` tag:  
```mdx
import image from './images/1.png';

<img
  alt="example"
  src={image}
/>
```

## Available MDX components  
The following Carbon MDX components are available for use in documentation:  

- `H1`, `H2`, `H3`, `H4`, `H5`, `H6`  
- `P`  
- `OL` (Ordered List), `UL` (Unordered List), `LI` (List Item)  
- `Accordion`, `AccordionItem`
- `AnchorLink`, `AnchorLinks`
- `ArticleCard`
- `Aside`
- `Blockquote`
- `Button`
- `Caption`
- `CardGroup`
- `Code`, `InlineCode`
- `DoDont`, `DoDontRow`
- `Divider`
- `GifPlayer`
- `Grid`
- `ImageWrapper`
- `InlineNotification`
- `Layer`
- `Link`
- `MiniCard`
- `PageDescription`
- `PageTable`, `Column`, `Row`
- `Preview`
- `ResourceCard`
- `StorybookDemo`
- `Tabs`, `Tab`
- `Title`
- `Video`

## Unsupported MDX components  
The following component are **not** currently supported in this documentation, although they may be available in Carbon websites:

- `A11yStatus`
